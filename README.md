# Predicting Behavior Composition from Steering-Vector Geometry

Code for the workshop paper *Predicting Behavior Composition from Steering-Vector Geometry*. We extract response-averaged difference-of-means steering vectors for eight persona/style behaviors in `Llama-3.1-8B-Instruct`, validate each direction individually (LLM judge + multiple-choice log-prob), then add every pair of vectors under two controlled injection schemes (normalized-sum and projection-controlled) and ask whether the pairwise geometry predicts the joint outcome, decomposed into a behavioral magnitude and a directional residual (reinforcement vs. suppression). `src/` holds the importable library (extraction, inference/steering, geometry, composition); `scripts/` holds one runnable driver per pipeline stage (extraction → validation → layer selection → composition → semantic control → human-eval); `analysis/rq1/` runs the headline statistic — an exact `8!` node-permutation (MRQAP) regression that reproduces the paper's main and robustness tables directly from the frozen frame in `analysis/rq1_consolidated/` (`cd analysis/rq1 && python nodeperm_analysis.py`, no GPU or API needed); `data/` and `external/` hold the multiple-choice datasets and trait artifacts; `slurm/` holds the cluster wrappers. Behavior scoring uses an OpenAI `gpt-4.1-mini` judge (set `OPENAI_API_KEY` in a `.env`); generation/extraction need a GPU, see `requirements*.txt` for the environment split. Trait artifacts for six behaviors are adapted from the Persona Vectors release (Chen et al., 2025, https://github.com/safety-research/persona_vectors); `confidence` and `formality` were generated with the same prompt template.
