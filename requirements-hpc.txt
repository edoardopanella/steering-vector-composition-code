# ============================================================
# HPC cluster environment (Linux + CUDA)
# ============================================================
# For full experimental runs on the university supercomputer.
# Assumes Linux x86_64 with NVIDIA GPUs.
#
# Install order:
#   1. Check CUDA version available on the cluster:
#        nvidia-smi
#        nvcc --version
#   2. Load appropriate modules (cluster-specific, e.g.):
#        module load python/3.11
#        module load cuda/12.1
#   3. python3.11 -m venv venv
#   4. source venv/bin/activate
#   5. pip install --upgrade pip
#   6. Install PyTorch matching the cluster's CUDA version:
#
#      For CUDA 12.1 (recommended if available):
#        pip install torch==2.5.1 torchvision==0.20.1 \
#          --index-url https://download.pytorch.org/whl/cu121
#
#      For CUDA 11.8:
#        pip install torch==2.5.1 torchvision==0.20.1 \
#          --index-url https://download.pytorch.org/whl/cu118
#
#   7. pip install -r requirements-hpc.txt
#   8. pip install -r requirements.txt
#
# GPU memory requirements for Llama-3.1-8B:
#   - fp16 / bf16: ~16 GB VRAM (A100, RTX 4090, H100 all fine)
#   - 8-bit with bitsandbytes: ~10 GB
#   - 4-bit with bitsandbytes: ~5 GB (fits on a single RTX 3090)
#
# If your cluster has smaller GPUs (e.g., RTX 3080, V100 16GB),
# use bitsandbytes quantization.
# ============================================================

# Quantization support for fitting 8B models on smaller GPUs
